# Screen Spec

## 1. Home / Travel Planner 화면

### 화면 목적
사용자가 여행 조건을 입력하고 AI 기반 추천 일정을 생성한다.

### 주요 구성요소
- 서비스 제목
- 서비스 설명 문구
- 목적지 입력 필드
- 여행 일정 입력 필드
- 동반자 선택 필드
- 여행 컨셉 선택 필드
- 추천 생성 버튼
- 필수 입력값 안내 메시지

### 레이아웃 규칙
- 상단에는 서비스 제목과 간단한 설명을 배치한다.
- 본문에는 여행 조건 입력 폼을 카드 형태로 배치한다.
- 추천 결과는 입력 폼 아래에 카드 3개로 표시한다.
- 모바일에서는 카드가 세로로 쌓이도록 한다.

### 상호작용
- 필수 항목이 모두 입력되면 추천 생성 버튼이 활성화된다.
- 추천 생성 버튼을 클릭하면 추천 결과 영역이 표시된다.
- 필수 항목이 비어 있으면 안내 메시지를 표시한다.

### 접근성
- 각 입력 필드에는 명확한 label을 제공한다.
- 추천 생성 버튼은 accessible name을 가진다.
- 제목 계층은 h1, h2, h3 순서를 유지한다.

## 2. Recommendation Result 화면

### 화면 목적
사용자가 3가지 테마별 여행 일정을 비교하고 선택할 수 있게 한다.

### 주요 구성요소
- 추천 결과 제목
- 3개의 테마별 여행 카드
- 각 카드의 테마명
- 각 카드의 설명
- 주요 장소 목록
- 지도 또는 동선 대체 영역
- 인플루언서 코스 추가 버튼

### 상태
- 기본 상태: 추천 결과가 아직 없음
- 로딩 상태: 추천 생성 중
- 결과 상태: 3개의 추천 카드 표시
- 오류 상태: 필수 입력값 누락 안내

### 스타일 규칙
- 전체적으로 깔끔한 여행 서비스 느낌을 유지한다.
- 카드형 UI를 사용한다.
- 버튼은 눈에 잘 띄게 배치한다.
- 지도 API 없이도 지도 대체 영역을 표시한다.
