# Acceptance Criteria

## AC-01 맞춤 여행 코스 생성

Given 사용자가 목적지, 일정, 여행 컨셉을 모두 입력한 상태에서  
When 코스 생성 버튼을 클릭하면  
Then 5초 이내에 3가지 서로 다른 테마의 추천 일정표가 출력되어야 한다.

Priority: High

---

## AC-02 지도 동선 표시

Given 추천 코스가 생성된 상태에서  
When 사용자가 추천 결과 화면을 확인하면  
Then 추천 일정과 함께 지도 또는 지도 대체 영역에 여행지 마커와 동선이 표시되어야 한다.

Priority: High

---

## AC-03 목적지 필수 입력값 검증

Given 목적지를 입력하지 않은 상태에서  
When 코스 생성 버튼을 클릭하면  
Then "목적지를 선택해주세요" 안내 메시지가 표시되고 다음 단계 진행이 차단되어야 한다.

Priority: High

---

## AC-04 여행 일정 필수 입력값 검증

Given 일정을 입력하지 않은 상태에서  
When 코스 생성 버튼을 클릭하면  
Then "여행 일정을 선택해주세요" 안내 메시지가 표시되고 다음 단계 진행이 차단되어야 한다.

Priority: High

---

## AC-05 여행 컨셉 필수 입력값 검증

Given 여행 컨셉을 선택하지 않은 상태에서  
When 코스 생성 버튼을 클릭하면  
Then "여행 컨셉을 선택해주세요" 안내 메시지가 표시되고 다음 단계 진행이 차단되어야 한다.

Priority: High

---

## AC-06 테마별 추천

Given 사용자가 여행 컨셉을 선택한 상태에서  
When 코스 생성 버튼을 클릭하면  
Then 선택한 컨셉에 맞는 테마 중심의 여행 코스가 우선 추천되어야 한다.

Priority: High

---

## AC-07 동반자 기반 추천

Given 사용자가 동반자 유형을 선택한 상태에서  
When 코스 생성 버튼을 클릭하면  
Then 동반자 유형에 적합한 장소와 이동 동선이 반영된 일정이 추천되어야 한다.

Priority: Medium

---

## AC-08 일정표 표시

Given 추천 코스가 생성된 상태에서  
When 사용자가 추천 결과 화면을 확인하면  
Then Day 1, Day 2 형식으로 시간대별 여행 일정이 표시되어야 한다.

Priority: High

---

## AC-09 지도 동선 순서 표시

Given 추천 코스가 생성된 상태에서  
When 사용자가 지도 영역을 확인하면  
Then 추천 장소들이 방문 순서대로 표시되고 선으로 연결된 것처럼 보여야 한다.

Priority: High

---

## AC-10 인플루언서 코스 연동

Given 인플루언서 추천 코스가 존재하는 상태에서  
When 사용자가 인플루언서 코스 추가 버튼을 클릭하면  
Then 해당 코스가 사용자의 일정표에 추가되었다는 상태가 표시되어야 한다.

Priority: High

---

## AC-11 코스 비교

Given 3가지 추천 코스가 생성된 상태에서  
When 사용자가 추천 결과 화면을 확인하면  
Then 각 코스의 테마, 예상 소요시간, 주요 장소, 추천 이유를 비교할 수 있어야 한다.

Priority: Medium

---

## AC-12 오류 처리

Given 추천 결과 생성 중 오류가 발생한 상태에서  
When 시스템이 추천 결과를 불러오지 못하면  
Then "추천 코스를 불러오지 못했습니다. 다시 시도해주세요" 메시지가 표시되어야 한다.

Priority: Medium
